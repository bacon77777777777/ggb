# 吉吉比（GGB）前後台測試案例（Test Cases）

> 版本 v1.1　｜　文件日期：2026-07-07　｜　撰寫：AI 助手 ＋ 行政組
> 用途：**首發前驗收基準**。前台以 Playwright E2E 覆蓋為目標，後台目前無自動化測試框架，先以人工驗收，逐步補 API 層測試。
>
> **自動化現況**（v1.1 更新）：
> - 後台 53 個 Vitest 測試（unit + API mock）已建立，`npm test` 可跑
> - 前台新增 sold-out.spec.ts、product-info.spec.ts
> - 詳細環境設定見 `03_測試環境設定.md`

## 使用說明

- **優先級**：P0 = 上線必須（金流、抽獎、代幣正確性）；P1 = 重要驗收；P2 = 加分項。
- **測試結果欄**：✅ 通過 / ❌ 失敗 / ⬜ 未測。
- **自動化欄**：🤖 已有自動化測試 / 📋 僅人工 / 🚧 待補
- **測試環境**：ECPay 使用測試帳號；DB 使用 seed 資料（reset_and_seed.sql）；測試帳號需排除 is_bot 標記。

## 自動化建議（優先順序）

首發前建議至少以下 P0 案例自動化：

- **前台**：B 組（金流）+ C 組（抽獎/代幣一致性）→ Playwright E2E
- **後台**：K 組（財務）+ M 組（GB哥）→ Vitest API 層

**現已完成（v1.1）**：
- `npm test`（後台）：53 個 Vitest 測試（unit + API mock）
- `npm run test:e2e`（前台）：5 個 Playwright 測試（navigation, PWA, market, sold-out, product-info）

**後台目前無測試框架，建議導入 API 整合測試**（可用 Playwright request 或 vitest + supertest），優先覆蓋有金流和代幣的路由。
