## 二、後台測試（Vitest）

### 2.1 安裝與執行

```bash
cd backend
npm install               # 已含 vitest, @vitest/coverage-v8
npm test                  # 跑一次（CI 模式）
npm run test:watch        # 開發時 watch 模式
npm run test:coverage     # 輸出覆蓋率報告
```

### 2.2 設定檔

`backend/vitest.config.ts` — 已設定：
- path alias `@/` → 根目錄
- 內建測試環境變數（不需 `.env`）
- coverage 排除 gbBro.ts / csAgent.ts（LLM 不適合 unit test）

### 2.3 測試目錄結構

```
backend/tests/
  setup.ts                   # 全域 console.* mock（抑制測試雜訊）
  helpers/

  api/
```

### 2.4 外部服務 Mock 策略

| 服務 | Mock 方式 | 說明 |

### 3.2 Supabase Mock 方式

所有 E2E 測試透過 `page.route('**/*supabase.co/rest/v1/**', ...)` 攔截 API 呼叫，回傳 fixture JSON。**不需要測試 Supabase 帳號**，不污染 production DB。

```typescript
await page.route('**/*supabase.co/rest/v1/**', async (route) => {
  const url = route.request().url();
  if (url.includes('/products') && url.includes('id=eq.9999')) {
    await route.fulfill({ status: 200, contentType: 'application/json',
      body: JSON.stringify(mockProduct) });
    return;
  }
  // fallback
  await route.fulfill({ status: 200, contentType: 'application/json', body: '[]' });
});
```

### 3.4 ECPay Callback 模擬（本機測試用）

ECPay sandbox **不會打本機 localhost**，需用 helper 直接打 API route：

```bash
# 在 backend 目錄手動觸發測試 callback
curl -X POST http://localhost:3001/api/payment/ecpay/callback \
  -F "MerchantID=2000132" \
  -F "MerchantTradeNo=TP20260101001" \
  -F "TradeNo=ECPAYTRADE001" \
  -F "TradeAmt=100" \
  -F "PaymentType=Credit_CreditCard" \
  -F "RtnCode=1" \
  -F "RtnMsg=交易成功" \
  -F "TradeDate=2026/01/01 00:00:00" \
  -F "SimulatePaid=0" \
  -F "CheckMacValue=<用 generateCheckMacValue 計算>"
```

或在 Vitest 中用 `makeEcpayCallback()` helper（已自動計算 CheckMacValue）。

## 五、CI/CD 整合

### 5.1 GitHub Actions

`.github/workflows/test.yml` — 每次 push 或 PR 到 main 自動觸發：

| Job | 觸發條件 | 時間估計 |
|---|---|---|
| `backend-unit` | 任何 push/PR | ~30s |
| `frontend-e2e` | 任何 push/PR | ~2min（含 Chromium 啟動） |

### 5.2 必要 Secrets（非敏感值可直接寫死，敏感值設定 GitHub Secrets）

| Secret | 說明 | 是否敏感 |
|---|---|---|
| `ADMIN_SESSION_SECRET` | 後台 session 簽章 | ✅ 設 GitHub Secret |
| `ECPAY_HASH_KEY/IV` | ECPay staging 測試值 | 可用 staging 公開值 |
| `LINE_CHANNEL_SECRET` | LINE 簽章驗證 | ✅ 設 GitHub Secret（可用假值跑 mock 測試）|
