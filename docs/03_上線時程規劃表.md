## 三、S2 正式金流切換清單（ECPay）

拿到正式 MerchantID / HashKey / HashIV 後，在 Vercel → ggb-backend → Environment Variables 更新：

| 變數 | 說明 |
|---|---|
| `ECPAY_MERCHANT_ID` | 正式商家編號（取代測試 3002607） |
| `ECPAY_HASH_KEY` | 正式 HashKey |
| `ECPAY_HASH_IV` | 正式 HashIV |
| `ECPAY_API_URL` | `https://payment.ecpay.com.tw/Cashier/AioCheckOut/V5` |
| `ECPAY_LOGISTICS_API_URL` | `https://logistics.ecpay.com.tw/Express/Create` |
| `ECPAY_LOGISTICS_MAP_URL` | `https://logistics.ecpay.com.tw/Express/map` |

> 金流與物流共用同一組 MerchantID / HashKey / HashIV，無需另設 `ECPAY_LOGISTICS_MERCHANT_ID`。
> 前台（ggb-frontend）不需異動任何變數。更新後 Redeploy 後台即生效。
> **切換後務必做正式小額實測**（真刷一筆小額並確認補幣、對帳、可退款）。

## 四、首發檢查清單（M1 Go / No-Go）

**金流與代幣（全 ✅ 才可上線）**

- [ ] 正式金流小額實測：下單→付款→補幣→對帳一致

## 五、負載與緊急應變

**分階段擴容**（依 DEVLOG 規劃）：

| 階段 | 觸發條件 | 動作 |
|---|---|---|
| 現況（0–500 會員） | — | Supabase/Vercel/Upstash/Sentry 皆 Free |
| 成長期（500–2,000） | 同時在線 >100 或 DB timeout | 升 Supabase Pro（$25/mo）、Vercel Pro（$20/mo） |
| 高峰活動（單日 UV >1,000） | 大型限時活動前 | 預先請 Supabase 提升連線、跑 k6 壓測 |

**負載異常處理流程**：Sentry 出現大量 timeout → 查 Supabase 連線數 → >80% 立即升 Pro → 仍無法解決則 feature flag 一鍵關閉轉蛋。
